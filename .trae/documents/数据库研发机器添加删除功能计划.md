# 数据库和研发机器添加删除功能实现计划

## 需求说明

配合已开发的权限管理系统，为数据库实例和研发机器页面添加：
- 添加功能
- 删除功能
- 根据用户权限（admin或can_edit/can_delete）控制按钮可见性

## 实现内容

### 1. 后端 (backend)

**schemas.py**：
- 新增 `DevMachineCreate` schema（研发机器创建数据）
- 新增 `DevMachineUpdate` schema（研发机器更新数据，可选）
- 新增 `DbInstanceCreate` schema（数据库实例创建数据）
- 新增 `DbInstanceUpdate` schema（数据库实例更新数据，可选）

**resources.py**：
- 新增 `POST /api/dev-machines` - 创建研发机器
- 新增 `DELETE /api/dev-machines/{machine_id}` - 删除研发机器
- 新增 `POST /api/db-instances` - 创建数据库实例
- 新增 `DELETE /api/db-instances/{instance_id}` - 删除数据库实例
- 所有新增接口都要有权限检查（admin或can_edit/can_delete）

### 2. 前端 (frontend)

**app.js**：
- 更新 `renderDevMachinesTable`：
  - 添加"添加"按钮到header actions
  - 在表格每行添加"删除"按钮
  - 根据权限控制按钮可见性
- 新增 `openAddDevMachineModal` - 打开研发机器添加弹窗
- 新增 `addDevMachine` - 提交添加研发机器
- 新增 `deleteDevMachine` - 删除研发机器
- 更新 `renderDbInstancesTable`：
  - 添加"添加"按钮到header actions
  - 在表格每行添加"删除"按钮
  - 根据权限控制按钮可见性
- 新增 `openAddDbInstanceModal` - 打开数据库添加弹窗
- 新增 `addDbInstance` - 提交添加数据库
- 新增 `deleteDbInstance` - 删除数据库

**style.css**：
- 添加需要的表格操作列样式

## 权限控制

- 查看权限：非访客用户
- 添加权限：admin 或 has can_edit
- 删除权限：admin 或 has can_delete

## 实现步骤

1. 更新 schemas.py 添加创建/更新数据结构
2. 更新 resources.py 添加后端 API
3. 更新前端 app.js 添加 UI 交互
4. 更新前端 app.js 更新表格渲染
5. 测试功能
