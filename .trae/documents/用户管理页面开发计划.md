# 管理员用户操作页面开发计划

## 功能需求

开发专门的管理员用户操作页面，支持：
1. 管理员管理其他用户的角色（普通用户/管理员）
2. 普通用户可设置指定分类的访问权限
3. 普通用户可设置编辑和删除权限

## 数据库设计

### 现有表结构
- `users` 表已有 `role` 字段 (guest/registered/admin)
- `user_permissions` 表已有权限存储结构

### 权限类型扩展
权限类型新增：
- `view` - 查看权限
- `edit` - 编辑权限
- `delete` - 删除权限

## 文件变更列表

### 后端
- `backend/routers/admin.py` - 新增用户管理页面数据接口
  - 获取用户列表（含完整权限信息）
  - 更新用户角色
  - 设置用户分类权限（批量）
  - 设置用户操作权限（edit/delete）

### 前端
- `frontend/index.html` - 添加用户管理入口（管理员可见）
- `frontend/js/app.js` - 添加用户管理页面渲染和交互
- `frontend/css/style.css` - 添加用户管理页面样式

## 实现步骤

### Phase 1: 后端 API 扩展
1. 更新 `GET /api/admin/users` 返回完整用户信息
2. 新增 `PUT /api/admin/users/{user_id}/categories` - 批量设置用户分类权限
3. 新增 `PUT /api/admin/users/{user_id}/actions` - 设置用户操作权限

### Phase 2: 前端页面
1. 在侧边栏添加「用户管理」菜单（管理员可见）
2. 用户列表页面展示：
   - 用户名、邮箱、角色
   - 分类权限复选框列表
   - 操作权限开关（编辑/删除）
3. 用户编辑功能：
   - 修改用户角色
   - 勾选可访问的分类
   - 开启/关闭编辑和删除权限
4. 保存用户设置

### Phase 3: 权限生效
1. 更新权限检查逻辑 `deps.py`
2. 前端根据权限动态显示操作按钮
