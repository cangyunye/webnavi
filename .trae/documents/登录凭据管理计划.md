# 登录凭据管理功能实现计划

## 功能需求

1. 数据库新增表存储研发机器和数据库实例的登录凭据
2. 只有管理员可以查看和管理这些凭据
3. 后台可以添加、编辑、删除、查看这些凭据信息

## 数据库设计

### 新增表

#### 1. `credentials` 表 - 登录凭据表
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| resource_type | VARCHAR(50) | 资源类型：dev_machine 或 db_instance |
| resource_id | INT | 资源 ID（关联 dev_machines 或 db_instances） |
| username | VARCHAR(100) | 登录用户名 |
| password | VARCHAR(255) | 登录密码（加密存储） |
| description | TEXT | 描述信息 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

## 文件变更列表

### 后端
- `sql/init.sql` - 新增 credentials 表，插入示例数据

### 前端
- `frontend/index.html` - 添加凭据管理界面入口

## 实现步骤

### Phase 1: 数据库
1. 更新 `sql/init.sql` 创建 credentials 表
2. 为现有研发机器和数据库实例添加示例凭据数据

### Phase 2: 后端模型
1. 更新 `backend/models.py` 添加 Credential 模型
2. 建立与 DevMachine 和 DbInstance 的关联
3. 更新 `backend/schemas.py` 添加相关 schemas

### Phase 3: 后端 API
1. 在 `routers/admin.py` 中添加凭据管理接口：
   - `GET /api/admin/credentials` - 获取凭据列表（含资源信息）
   - `GET /api/admin/credentials/{id}` - 获取凭据详情
   - `POST /api/admin/credentials` - 添加凭据
   - `PUT /api/admin/credentials/{id}` - 更新凭据
   - `DELETE /api/admin/credentials/{id}` - 删除凭据
2. 添加按资源类型筛选功能
3. 添加密码加密存储（可选使用 passlib）

### Phase 4: 前端
1. 添加侧边栏或页面入口（管理员可见）
2. 凭据列表页面（表格展示，含资源信息）
3. 添加/编辑凭据的模态框
4. 凭据管理功能实现
