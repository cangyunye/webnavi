## 导航系统

### 侧边栏导航

- **左侧侧边栏**：显示所有分类
- **可收起/展开**：点击顶部箭头按钮可收起侧边栏
- **图标模式**：收起后只显示图标，节省空间
- **活动状态**：当前选中的分类会高亮显示
- **权限标识**：无权限的分类会变半透明

### 分类列表

默认 8 个分类：

| 排序 | 名称 | 图标 | 类型 | 访客可访问 |
|------|------|------|------|-----------|
| 1 | 学习 | 📚 | 网站资源 | ✅ |
| 2 | 数据库 | 🗄️ | 运维表格（数据库实例） | ❌ |
| 3 | 研发机器 | 🖥️ | 运维表格（服务器） | ❌ |
| 4 | AI | 🤖 | 网站资源 | ✅ |
| 5 | 测试 | 🧪 | 网站资源 | ✅ |
| 6 | 软件资源 | 💿 | 网站资源 | ✅ |
| 7 | 运维 | ⚙️ | 网站资源 | ❌ |
| 8 | 工具 | 🛠️ | 网站资源 | ✅ |

### 主页卡片

- **卡片式展示**：每个分类一个卡片，带 emoji 图标
- **渐变色设计**：每个卡片有独特的渐变背景
- **悬停效果**：鼠标悬停时卡片上浮并显示阴影
- **点击跳转**：点击卡片直接跳转到对应分类页面
- **响应式布局**：根据屏幕宽度自动调整卡片列数

### 分类行为

- **网站资源类**（学习、AI、软件资源、测试、运维、工具）：卡片网格展示，支持添加/删除
- **运维表格类**（数据库、研发机器）：表格展示，支持筛选/添加/修改/删除

## 资源管理

### 资源卡片

- **卡片式布局**：每个网站资源一个卡片
- **图标展示**：支持 emoji 图标
- **链接跳转**：点击卡片在新标签页打开链接
- **描述信息**：显示资源描述
- **删除功能**：注册用户可见右上角删除按钮

### 添加资源

- **添加卡片**：在分类页面最后显示一个"➕"添加卡片
- **弹出表单**：点击后弹出添加资源表单
- **表单字段**：资源名称 / 资源链接 / 资源描述（可选）
- **权限控制**：只有注册用户和管理员可以添加

### 删除资源

- **删除按钮**：悬停在卡片上时右上角显示删除按钮
- **确认提示**：点击删除会弹出确认对话框
- **权限控制**：只有注册用户和管理员可以删除

### 工具分类默认资源

|------|------|------|
| wttr.in | https://wttr.in | 命令行风格的天气预报工具 |
| Public APIs | https://github.com/public-apis/public-apis | 开源 API 集合 |
| Papers With Code | https://paperswithcode.com | 论文+数据集+代码汇总 |
| Hugging Face Datasets | https://huggingface.co/datasets | 开源机器学习数据集仓库 |

## 研发环境

### 研发机器管理

- **表格展示**：运维风格的表格
- **信息字段**：名称 / IP:端口 / 环境 / 状态 / 负责人 / 组织 / CPU / 内存 / 操作系统
- **状态标识**：绿色在线，灰色离线
- **环境徽章**：蓝色开发、紫色测试、红色生产
- **筛选**：按环境（dev/test/prod）和状态（在线/离线）
- **CRUD**：表格顶部「添加研发机器」；操作列修改/删除；右侧滑出面板
- **权限**：管理员或 `can_edit` 用户可添加/修改；管理员或 `can_delete` 用户可删除

### 数据库实例管理

- **表格展示**：运维风格的表格
- **信息字段**：名称 / 类型 / 版本 / 地址 / 环境 / 状态 / 负责人 / 组织 / 字符集
- **支持类型**：MySQL, PostgreSQL, Redis, MongoDB, Elasticsearch, ClickHouse
- **筛选**：按类型和环境
- **CRUD**：与研发机器相同模式
- **权限**：与研发机器相同

### 组织与责任人

- **组织管理**：预置 6 个组织（基础架构部、前端研发组等）
- **责任人管理**：预置 8 个责任人，可关联组织和资源
- 组织和责任人在添加/修改研发机器和数据库时可选择

## 用户与权限

### 用户角色

| 角色 | 说明 |
|------|------|
| **访客 (guest)** | 无需登录，仅可访问题目为 "学习"、"AI"、"软件资源"、"测试"、"工具" 的分类，不可添加/删除 |
| **注册用户 (registered)** | 登录后可访问所有分类，可添加/删除资源 |
| **管理员 (admin)** | 完整权限，可访问管理后台 |

### 用户认证

- **JWT Token** 认证（默认 7 天有效期）
- **支持 API Key** 认证（与 JWT 并存）
- 未提供 Token 时自动降级为访客（id=0, role=guest）

#### 注册
- 用户名（唯一，必填）、密码（至少 6 位，必填）、邮箱（可选，唯一）
- 注册后自动登录，返回 JWT Token

#### 登录
- 用户名+密码登录，返回 JWT Token
- 访客一键登录（`POST /api/auth/guest`）

### 权限体系

- **权限类型**：`can_edit`（可添加/修改）、`can_delete`（可删除）
- **分类权限**：可为注册用户设置允许访问的分类白名单
- 未设置分类权限的注册用户默认可访问全部分类
- 访客固定可访问：学习、AI、软件资源、测试、工具

## 管理后台

仅管理员（`role=admin`）可访问，入口在侧边栏底部。

### 用户列表

- 全部用户表格展示，排除 id=0 的访客虚拟用户
- 顶部显示「共 N 个用户」统计
- 每行显示：用户名 / 邮箱 / 角色 / 状态 / 可编辑 / 可删除 / 分类权限 / 操作

### 用户管理操作

- **修改角色**：下拉切换 guest / registered / admin
- **启用/禁用**：开关切换，禁用后无法登录
- **编辑权限**：开关切换 `can_edit` 和 `can_delete`
- **分类权限**：弹窗勾选允许访问的分类
- **重置密码**：弹窗输入新密码 + 确认密码，含长度校验和一致性校验
- **添加用户**：管理员直接创建用户，指定用户名 / 邮箱 / 密码 / 角色（普通用户或管理员）

## API Key 管理

所有已登录用户（非访客）可用，入口在侧边栏"API Key 管理"。

### 功能

- **创建 API Key**：指定名称和可选过期时间，生成 `sk_` 前缀的密钥
- **列表查看**：显示名称、前缀、状态、最后使用时间、创建时间
- **删除**：删除后不可恢复
- **使用日志**：查看每个 Key 的调用记录（时间、方法、端点、IP、状态码）

### 安全

- API Key 使用 bcrypt 加密存储，数据库仅存哈希
- 完整 Key 仅在创建时展示一次，不会再次显示
- 支持设置过期时间
- 用户只能管理自己创建的 Key
- API Key 可用于调用 `/api/v1/nodes` 接口，替代 JWT 登录

## CMDB 节点管理

### 节点 CRUD

- **路由**：`/api/v1/nodes`
- **认证**：JWT 或 API Key（访客不可用）
- **支持的过滤**：名称、分组、标签、状态
- **分页**：page / page_size（默认 100）
- **响应格式**：`{ code: 0, message: "success", data: { total, page, page_size, items } }`

### 节点字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 节点名称 |
| hostname | string | 主机名 |
| address | string | IP 地址 |
| port | int | SSH 端口（默认 22） |
| user | string | 登录用户（默认 root） |
| status | string | active / inactive |
| groups | string[] | 分组标签 |
| labels | object | 自定义标签 |
| ssh_key | string | SSH 密钥路径 |
| ssh_password | string | SSH 密码 |
| proxy_jump | string | 跳板机 |

## 枚举管理

动态枚举/下拉选项系统，管理员可管理各业务模块的选项数据。

### 已注册枚举类型

- **environment**：环境类型（dev/test/prod/staging/sandbox/local/uat/dr）
- **db_type**：数据库类型（MySQL/PostgreSQL/Redis 等 16 种）
- **resource_type**：资源类型（研发机器、数据库实例）
- **organization**：组织列表（10 种）
- **node_status**：节点状态（active/inactive/maintenance）
- **general_status**：通用状态（online/offline/pending/error）
- **user_role**：用户角色
- **permission_type**：权限类型（view/edit/delete/manage）
- **os_type**：操作系统
- **alert_level**：告警级别
- **log_level**：日志级别

### API

| 端点 | 说明 |
|------|------|
| `GET /api/enum-items` | 获取枚举项列表 |
| `GET /api/enum-items/types` | 获取所有枚举类型名称 |
| `GET /api/enum-items/{type}/options` | 获取下拉选项（value/label） |
| `GET /api/enum-items/{type}/options-full` | 获取完整选项（含颜色、图标） |
| `POST /api/enum-items` | 创建枚举项（管理员） |
| `PUT /api/enum-items/{id}` | 更新枚举项（管理员） |
| `DELETE /api/enum-items/{id}` | 删除枚举项（管理员） |
| `POST /api/enum-items/{type}/reorder` | 重新排序（管理员） |

## 凭据管理

仅管理员可访问，入口在侧边栏底部"凭据管理"。

### 功能

- 存储研发机器和数据库实例的登录用户名和密码
- 明文显示密码（实际使用场景需要可读）
- 关联具体资源（下拉选择机器或数据库）
- 支持添加、编辑、删除

### 与 API Key 的区别

| 维度 | API Key | 凭据 |
|------|---------|------|
| 用途 | 外部程序接口认证 | 存储资源登录密码 |
| 可见范围 | 用户自己 | 仅管理员 |
| 存储方式 | bcrypt 哈希 | 明文 |
| 可登录 | 否（仅 API 调用） | 是（SSH/数据库登录） |

## 前端界面

### 整体风格

- **浅色主题**：简洁清爽
- **卡片式布局**：现代流行的设计风格
- **响应式设计**：适配桌面/平板/移动
- **流畅动画**：悬停和交互时有平滑动画

### 侧边栏

- **展开模式**：约 200px，显示图标+文字
- **收起模式**：约 60px，仅显示图标
- **移动端**：默认收起状态

### 模态框

- **登录/注册**：用户名+密码登录、注册表单
- **添加资源**：资源名称/链接/描述
- **添加用户**：管理员创建用户（用户名/邮箱/密码/角色）
- **重置密码**：新密码+确认密码
- **设置分类权限**：勾选分类

### 右侧滑出面板

用于研发机器和数据库实例的添加/修改：
- 从右侧平滑滑出
- 完整的编辑表单
- 修改模式自动预填现有数据
- 取消关闭 / 保存提交

### 响应式适配

- **桌面端**：3-4 列卡片
- **平板端**：2-3 列卡片
- **移动端**：1-2 列卡片
