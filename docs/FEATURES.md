## 导航系统

### 侧边栏导航

- **左侧侧边栏**：显示所有分类
- **可收起/展开**：点击顶部箭头按钮可收起侧边栏
- **图标模式**：收起后只显示图标，节省空间
- **活动状态**：当前选中的分类会高亮显示
- **权限标识**：无权限的分类会变半透明

### 分类列表

默认 8 个分类：

| 排序 | 名称 | 图标 | 类型 | 公开（访客+注册用户） |
|------|------|------|------|---------------------|
| 1 | 学习 | 📚 | 网站资源 | ✅ |
| 2 | 数据库 | 🗄️ | 运维表格（数据库实例） | ❌ |
| 3 | 研发机器 | 🖥️ | 运维表格（服务器） | ❌ |
| 4 | AI | 🤖 | 网站资源 | ✅ |
| 5 | 测试 | 🧪 | 网站资源 | ✅ |
| 6 | 软件资源 | 💿 | 网站资源 | ✅ |
| 7 | 运维 | ⚙️ | 网站资源 | ❌ |
| 8 | 工具 | 🛠️ | 网站资源 | ✅ |

### 主页卡片

- **卡片式展示**：每个分类一个卡片，带 emoji 图标
- **渐变色设计**：每个卡片有独特的渐变背景
- **悬停效果**：鼠标悬停时卡片上浮并显示阴影
- **点击跳转**：点击卡片直接跳转到对应分类页面
- **响应式布局**：根据屏幕宽度自动调整卡片列数

### 分类行为

- **网站资源类**（学习、AI、软件资源、测试、运维、工具）：卡片网格展示，支持添加/删除
- **运维表格类**（数据库、研发机器）：表格展示，支持筛选/添加/修改/删除

## 资源管理

### 资源卡片

- **卡片式布局**：每个网站资源一个卡片，带 emoji 图标
- **链接跳转**：点击卡片在新标签页打开链接
- **描述信息**：显示资源描述
- **删除按钮**：有管理权限的用户悬停时可见右上角删除按钮

### 添加/删除资源

- 分类页面末尾显示"➕"添加卡片
- 弹窗表单：资源名称 / 资源链接 / 描述（可选）
- 权限控制：仅 学习导师、管理员 可管理学习/AI/软件资源/测试/工具

### 默认资源

**学习**：GitHub, Stack Overflow, MDN Web Docs, 菜鸟教程, LeetCode, Codecademy

**AI**：OpenAI, Hugging Face, Stable Diffusion, MidJourney, Claude, Perplexity

**软件资源**：VS Code, Docker, Notion, Figma, Chrome

**测试**：Postman, Selenium, Jest, JMeter, Swagger

**工具**：wttr.in, Public APIs, Papers With Code, Hugging Face Datasets, DevDocs

**运维**：Grafana, Prometheus, Jenkins, Ansible, ELK Stack

## 研发环境

### 研发机器管理

- **表格展示**：运维风格表格，含统计卡片（在线/待处理/故障/离线）
- **信息字段**：名称 / IP:端口 / 环境 / 状态 / 负责人 / 组织 / CPU / 内存 / 操作系统
- **状态**：online（在线）、offline（离线）、pending（待处理）、error（故障），彩色标识
- **环境**：从枚举表动态获取（dev/test/prod/staging/sandbox/local/uat/dr），自动展示
- **筛选**：按环境、状态、搜索关键词
- **分页**：客户端分页，默认 15/页，可选 30/50/100，`<` `>` 翻页
- **CRUD**：表格顶部添加；操作列修改/删除；右侧滑出面板
- **权限**：管理员和 ops_expert 可进行 CRUD

### 数据库实例管理

- **表格展示**：运维风格表格，与研发机器相同模式
- **信息字段**：名称 / 类型 / 版本 / 地址 / 环境 / 状态 / 负责人 / 组织 / 字符集
- **数据库类型**：从枚举表动态获取（MySQL/PostgreSQL/Redis/MongoDB 等 16 种）
- **统计/筛选/分页/CRUD/权限**：与研发机器相同

### 组织与责任人

- **组织管理**：预置 6 个组织（基础架构部、前端研发组等）
- **责任人管理**：预置 8 个责任人
- 添加/修改研发机器和数据库时可选择组织和责任人

## 用户与权限

### 用户角色

| 角色 | 可读 | 可管理 |
|------|------|--------|
| **访客 (guest)** | 学习、AI、软件资源、测试、工具 | — |
| **注册用户 (registered)** | 学习、AI、软件资源、测试、工具 | — |
| **学习导师 (learning_mentor)** | 全部分类 | 学习、AI、软件资源、测试、工具 |
| **运维专家 (ops_expert)** | 全部分类 | 运维、数据库、研发机器 |
| **管理员 (admin)** | 全部 + 管理后台 | 全部 |

### 用户认证

- **JWT Token** 认证（默认 7 天有效期）
- **支持 API Key** 认证（与 JWT 并存）
- 未提供 Token 时自动降级为访客（id=0, role=guest）
- 访客一键登录（`POST /api/auth/login`）
- 右上角有**退出按钮**，清除凭证后跳转首页

#### 注册
- 用户名（唯一，必填）、密码（至少 6 位，必填）、邮箱（可选，唯一）
- 注册后自动登录，返回 JWT Token

### 权限体系

- **`can_edit` / `can_delete`**：仅对访客和注册用户有效，通过管理后台开关控制
- **学习导师**和**运维专家**的编辑/删除权限由角色定义，不通过独立开关控制
- 访客和注册用户可访问：学习、AI、软件资源、测试、工具
- 角色变更仅管理员可在管理后台操作

## 管理后台

仅管理员（`role=admin`）可访问，入口在侧边栏底部。包含三个标签页。

### 用户管理

- 全部用户表格展示（排除 id=0 的访客虚拟用户）
- 顶部统计「共 N 个用户」
- 每行显示：用户名 / 邮箱 / 角色 / 状态 / 可编辑 / 可删除 / 操作
- **修改角色**：下拉切换 5 种角色
- **启用/禁用**：开关切换
- **编辑/删除权限**：访客和注册用户可开关；学习导师和运维专家显示"由角色控制"（不可手动变更）
- **重置密码**：弹窗输入新密码 + 确认密码
- **添加用户**：管理员直接创建用户，可指定角色（注册用户/学习导师/运维专家/管理员）

### API Key 管理

- 所有已登录用户（非访客）可用
- **创建**：指定名称和可选过期时间（7天/30天/90天/1年），生成 `sk_` 前缀密钥
- 完整 Key 仅在创建时展示一次，不会再次显示
- **查看列表**：名称、前缀、状态、最后使用时间、创建时间
- **删除**：删除后不可恢复
- **使用日志**：查看每个 Key 的调用记录

### 凭据管理

- 仅管理员可见
- 存储研发机器和数据库实例的登录用户名和密码（明文）
- 支持添加、编辑、删除

### 与 API Key 的区别

| 维度 | API Key | 凭据 |
|------|---------|------|
| 用途 | 外部程序接口认证 | 存储资源登录密码 |
| 可见范围 | 用户自己 | 仅管理员 |
| 存储方式 | bcrypt 哈希 | 明文 |
| 可登录 | 否（仅 API 调用） | 是（SSH/数据库登录） |

## CMDB 节点管理

### 节点 CRUD

- **路由**：`/api/v1/nodes`
- **认证**：JWT 或 API Key（访客不可用）
- **支持的过滤**：名称、分组、标签、状态
- **分页**：page / page_size（默认 100）
- **响应格式**：`{ code: 0, message: "success", data: { total, page, page_size, items } }`

### 节点字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 节点名称 |
| hostname | string | 主机名 |
| address | string | IP 地址 |
| port | int | SSH 端口（默认 22） |
| user | string | 登录用户（默认 root） |
| status | string | active / inactive |
| groups | string[] | 分组标签 |
| labels | object | 自定义标签 |
| ssh_key | string | SSH 密钥路径 |
| ssh_password | string | SSH 密码 |
| proxy_jump | string | 跳板机 |

## 枚举管理

动态枚举/下拉选项系统，管理员可管理各业务模块的选项数据。

### 已注册枚举类型

- **environment**：环境类型（dev/test/prod/staging/sandbox/local/uat/dr）
- **db_type**：数据库类型（MySQL/PostgreSQL/Redis 等 16 种）
- **resource_type**：资源类型（研发机器、数据库实例）
- **organization**：组织列表（10 种）
- **node_status**：节点状态（active/inactive/maintenance）
- **general_status**：通用状态（online/offline/pending/error）
- **user_role**：用户角色（guest/registered/learning_mentor/ops_expert/admin）
- **permission_type**：权限类型（view/edit/delete/manage）
- **os_type**：操作系统
- **alert_level**：告警级别
- **log_level**：日志级别

### API

| 端点 | 说明 |
|------|------|
| `GET /api/enum-items` | 获取枚举项列表 |
| `GET /api/enum-items/types` | 获取所有枚举类型 |
| `GET /api/enum-items/{type}/options` | 获取下拉选项（value/label） |
| `GET /api/enum-items/{type}/options-full` | 完整选项（含颜色、图标） |
| 其余 | 管理员管理 CRUD |

## 前端界面

### 整体风格

- **支持浅色/深色主题**切换（右上角 🌙/☀️ 按钮）
- 主题选择存储到 `localStorage`，所有页面统一
- **响应式设计**：适配桌面/平板/移动
- **5 个独立 HTML 页面**：首页 / 分类资源 / 运维管理 / 管理后台 / 登录

### 侧边栏

- **展开模式**：约 220px，显示图标+文字
- **收起模式**：约 60px，仅显示图标
- **移动端**：默认隐藏，点击汉堡菜单展开

### 模态框

- **登录/注册**：独立页面（login.html），含标签页切换
- **添加资源**：资源名称/链接/描述
- **添加用户**：管理员创建用户
- **重置密码**：新密码+确认密码

### 右侧滑出面板

用于研发机器和数据库实例的添加/修改：
- 从右侧平滑滑出
- 表单字段完整，环境选项从枚举动态获取
- 修改模式自动预填现有数据（含组织和责任人）
- 取消关闭 / 保存提交
